# Frontend Architecture: The Glass Window

The NIMS frontend is a Next.js (React) application built to be the "glass window" into the user's memories. The UI prioritizes a dynamic, highly responsive aesthetic, moving away from a traditional "grid of photos" toward an exploratory visual map.

## Key Design Philosophies
- **Dynamic Theming:** The application changes its dominant color palette based on the memory or identity you are currently viewing. If you view an album from a forest hike, the UI accentuates deep greens and earthy tones.
- **Glassmorphism:** We employ frosted glass effects prominently to give a sense of depth and fluidity, allowing memories in the background to bleed softly through panels.
- **Micro-Animations:** Every interaction (hover, scroll, click) introduces subtle motion to make the application feel alive.

## Application Structure

### 1. The Memory Vault (Gallery View)
The primary landing experience. 
- Implements infinite scrolling and eager-loading of low-res blurhashes before swapping to high-fidelity images.
- Images aren't just displayed; hovering over them reveals contextual metadata (who is in it, date, AI-generated caption) generated by the backend's `VlmService`.

### 2. Identity Explorer (Social Graph View)
Unlike traditional galleries, NIMS understands people as first-class citizens.
- Displays automatically generated avatars (crops) for individuals found across your memory vault.
- Provides a connection graph showing who frequently appears with whom, allowing you to traverse your memories via social links rather than pure chronology.

### 3. Neural Chat Assistant
A persistent, floating conversational interface to the system.
- Driven by Retrieval-Augmented Generation (RAG). As the user types, the Next.js frontend sends queries to the NestJS backend, which retrieves semantically similar memories.
- The chat bubble can display structured widgets (e.g., a mini-gallery of results) directly within the conversation stream, rather than just text.

## State Management and Data Fetching
- The frontend relies on robust caching strategies for images and embeddings to ensure instantaneous load times after the first fetch.
- API interactions with the NestJS backend are handled securely, retrieving atomic representations of the `ImageMemoryStore` to hydrate the UI.
